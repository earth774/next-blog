Cursor 3 ถูกวางตำแหน่งว่าเป็น "พื้นที่ทำงานแบบรวมศูนย์" สำหรับพัฒนาซอฟต์แวร์กับเอเจนต์ โดยอินเทอร์เฟซใหม่เน้นให้มองงานที่เอเจนต์ทำได้ชัดขึ้น สามารถเจาะลึกเมื่อต้องการ และรองรับการทำงานหลายรีโปในเลย์เอาต์เดียวกัน พร้อมการส่งต่อระหว่างเอเจนต์บนเครื่องและบนคลาวด์ได้ลื่นขึ้น รวมถึงสลับกลับไปใช้ Cursor IDE แบบเดิมได้ตลอดเวลา

---

## Cursor 3.0 เมื่อไหร่ และเกิดอะไรขึ้น

ตาม [Changelog 3.0](https://cursor.com/changelog/3-0) ของ Cursor เวอร์ชัน 3.0 เปิดใช้งานในวันที่ **2 เมษายน 2026** ส่วนรายละเอียดเชิงผลิตภัณฑ์และภาพรวมการออกแบบใหม่ อธิบายเพิ่มเติมในโพสต์ [Meet the new Cursor](https://cursor.com/blog/cursor-3)

## Agents Window: เอเจนต์หลายตัว หลายที่ทำงาน

**Agents Window** คืออินเทอร์เฟซใหม่ที่จัดโฟกัสรอบเอเจนต์ โดยรองรับการรันเอเจนต์ **หลายตัวแบบขนาน** ข้ามรีโปและสภาพแวดล้อมได้ — ทั้ง **บนเครื่อง (local)**, ใน **git worktree**, บน **คลาวด์**, และผ่าน **Remote SSH** ตามที่ระบุใน Changelog

การลองใช้งาน: อัปเกรด Cursor แล้วเปิดผ่าน **Command Palette** ด้วย **`Cmd+Shift+P` → "Agents Window"** (บน Windows/Linux ใช้ `Ctrl+Shift+P` เพื่อเปิด Command Palette ตามปกติของ Cursor) ตามที่ทั้งบล็อกและ Changelog แนะนำ คุณ **สลับกลับไป IDE ได้ทุกเมื่อ** หรือจะ **เปิดทั้งสองพร้อมกัน** ก็ได้

## หลาย workspace และแหล่งที่มาของเอเจนต์

อินเทอร์เฟซใหม่ถูกออกแบบให้รองรับ **หลาย workspace โดยธรรมชาติ** (multi-workspace) เพื่อให้ทั้งมนุษย์และเอเจนต์ทำงานข้ามรีโปต่าง ๆ ได้สะดวกขึ้น

เอเจนต์ทั้งหมด — ทั้งบนเครื่องและบนคลาวด์ — ปรากฏในแถบด้านข้าง รวมถึงเซสชันที่คุณเริ่มจาก **มือถือ, เว็บ, เดสก์ท็อป, Slack, GitHub และ Linear** ตามที่ระบุในบล็อก

## คลาวด์: เดโม สกรีนช็อต และประสบการณ์เดียวกับเว็บ

เอเจนต์บนคลาวด์สามารถ **สร้างเดโมและสกรีนช็อต** ของงานที่ทำให้คุณตรวจสอบได้ โดย Cursor ระบุว่านี่คือ **ประสบการณ์เดียวกับที่ [cursor.com/agents](https://cursor.com/agents)** แต่ถูก **ผสานเข้าในแอปเดสก์ท็อป**

## ส่งต่อระหว่างคลาวด์กับเครื่อง (Handoff)

บล็อกเน้นว่าการ **ย้ายเอเจนต์ระหว่างสภาพแวดล้อม** ทำได้เร็วขึ้น:

- **จากคลาวด์ → เครื่อง**: เมื่อต้องการแก้ไขและทดสอบบนเดสก์ท็อปของคุณเอง
- **จากเครื่อง → คลาวด์**: เพื่อให้งานยังรันต่อได้เมื่อออฟไลน์ หรือเพื่อไปทำงานถัดไป โดยเฉพาะงานที่รันนาน ๆ ที่มักถูกขัดจังหวะเมื่อปิดแล็ปท็อป

ในบล็อกมีการกล่าวถึง **Composer 2** ว่าเป็น **โมเดลเขียนโค้ดระดับ frontier ของ Cursor พร้อมขีดจำกัดการใช้งานที่สูง** — เหมาะกับการวนลูปเร็ว ๆ (ไม่มีการระบุราคาในหน้าที่อ้างอิง)

## จาก commit ไปจนถึง PR

มุมมอง **diffs ใหม่** ช่วยให้แก้ไขและรีวิวการเปลี่ยนแปลงได้เร็วขึ้นด้วย UI ที่เรียบง่าย และเมื่อพร้อมแล้วสามารถ **stage, commit และจัดการ PR** ได้ในมุมมองเดียวกันกับการรีวิว (ตามทิศทางที่บล็อกเน้นเรื่อง diffs และ PR)

## ความลึกของ IDE: ไฟล์ LSP และเบราว์เซอร์ในตัว

Cursor 3 ยังผสานจุดแข็งของ IDE เดิมไว้ด้วย:

- **ไฟล์เพื่อทำความเข้าใจโค้ด**: เปิดดูไฟล์ได้ตลอด และใช้ **go to definition** ในเอดิเตอร์พร้อม **LSP เต็มรูปแบบ**
- **เบราว์เซอร์ในตัว**: เปิด นำทาง และให้เอเจนต์ทำงานกับเว็บไซต์ในเครื่อง (local) ได้ — รายละเอียดเชิงเครื่องมืออยู่ในเอกสาร browser ของ Cursor ตามลิงก์ในบล็อก

## Marketplace ปลั๊กอิน MCP Skills และ Subagents

ในบล็อก Cursor ชวนให้ค้นหา **ปลั๊กอินหลายร้อยรายการ** บน Cursor Marketplace ที่ขยายความสามารถเอเจนต์ด้วย **MCP, skills, subagents** และอื่น ๆ ติดตั้งคลิกเดียว หรือตั้งค่า **[team marketplace](https://cursor.com/docs/plugins#team-marketplaces)** สำหรับปลั๊กอินภายในองค์กร

## Design Mode (จาก Changelog)

ใน Agents Window มี **Design Mode** สำหรับ **คอมเมนต์และเล็ง UI ในเบราว์เซอร์** เพื่อให้ฟีดแบ็กชัดขึ้นและวนลูปเร็วขึ้น โดยทางลัดคีย์บอร์ดที่ระบุใน Changelog ได้แก่:

- **`⌘+Shift+D`**: สลับเข้า/ออก Design Mode
- **`Shift` + ลาก**: เลือกพื้นที่
- **`⌘+L`**: เพิ่มองค์ประกอบเข้าแชท
- **`⌥` + คลิก**: เพิ่มองค์ประกอบเข้าช่องอินพุต

บน Windows/Linux ให้ตรวจคีย์ลัดที่สอดคล้องกับเครื่องของคุณในเอกสารล่าสุดของ Cursor

## Agent Tabs ใน Editor

**Agent Tabs** ให้ดูแชทหลายกระทู้พร้อมกันได้ ทั้งแบบ **เคียงข้าง** หรือ **เรียงเป็นตาราง**

### ใน Editor ตาม Changelog (4 รายการ)

- **`/worktree`**: สร้าง git worktree แยก เพื่อให้การเปลี่ยนแปลงเกิดแยกจากกิ่งหลัก
- **`/best-of-n`**: รันงานเดียวกันแบบขนานบนหลายโมเดล แต่ละตัวใน worktree แยก แล้วเปรียบเทียบผล
- **เลิกใช้ (deprecated)** วิธีเลือก worktree และ best-of-n แบบเดิมใน Editor
- **ถอด cloud agents ออกจาก Editor** — การจัดการ cloud agents ถูกผูกกับทิศทางอินเทอร์เฟซใหม่ตามบริบทที่ Changelog สื่อ

### Plugins และ MCP (สองรายการใน Changelog)

- **MCP Apps** รองรับ **structured content** เพื่อให้เอาต์พุตจากเครื่องมือหลากหลายและอ่านง่ายขึ้น
- **การนำเข้าปลั๊กอินบุคคลที่สาม**: ค่าเริ่มต้นสำหรับ Enterprise เมื่อยังไม่ตั้งค่า คือ **ปิด** — แต่ยัง **เคารพการ override ของ Admin** ที่ตั้งไว้อย่างชัดเจน

### Enterprise และทีม (สามรายการใน Changelog)

- **Audit logs**: เพิ่มชื่อกลุ่มในไดเรกทอรีให้อ่านได้โดยไม่ต้องไล่ lookup จาก ID
- **การตั้งค่าระดับทีมสำหรับ cloud agents**: จำกัดการสร้าง/แก้ไข/ลบ **team secrets** ให้เป็น **Admin** เท่านั้น
- **Enterprise Admin**: ควบคุมการปิด **"Made with Cursor" code attribution** ทั้งทีมได้ — การตั้งค่ารายบุคคลยังมีที่ Cursor Settings > Agent > Attribution

### Other Improvements (Changelog รวม 10 รายการ — ชี้ทางบางข้อ)

Changelog 3.0 รวมหมวด **Other Improvements** หลายข้อ; ตัวอย่างที่น่าสนใจสำหรับเวิร์กโฟลว์ประจำวัน เช่น **การเรนเดอร์ diff ไฟล์ใหญ่ที่เร็วและนุ่มนวลขึ้น**, **เครื่องมือ `Await`** ให้เอเจนต์รอคำสั่งเชลล์/ซับเอเจนต์ในพื้นหลังหรือรอเอาต์พุตเฉพาะ (เช่น "Ready" หรือ "Error"), และ **การปรับปรุงเบราว์เซอร์อัตโนมัติ** (ลดพื้นผิวเครื่องมือ คำสั่งชัดขึ้น มีคลิกตามพิกัดจากสกรีนช็อตเมื่อ DOM ไม่น่าเชื่อถือ) — รายการที่เหลืออ่านได้ครบใน Changelog

## ข้อควรทราบ

รายละเอียดผลิตภัณฑ์ การตั้งค่า และพฤติกรรมของ Cursor **อาจเปลี่ยนแปลงได้ตามการอัปเดต** ผู้อ่านควรตรวจสอบ **Changelog และเอกสารล่าสุด** ก่อนตัดสินใจใช้งานหรือวางนโยบายภายในองค์กร
